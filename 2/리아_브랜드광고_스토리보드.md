# 스토리보드 (기획 문서)
AI 기반 브랜드 광고 패키지 — 가상 브랜드 "리아(Ria)" 광고 영상 기획

> 본 문서는 B1-1 과제에서 제작한 CS 환불/교환 응대 메일 어시스턴트 "리아"를 가상 브랜드로 삼아, 숏폼 광고 영상을 기획·제작한 기록이다.

---

## 1. 브랜드 아이덴티티

- 브랜드명: 리아 (Ria)
- 타겟: 직접 환불/교환 응대 메일을 작성해야 하는 이커머스·소규모 쇼핑몰 CS 담당자
- 톤앤매너: 차분한 그레이/블루 톤, 미니멀, 신뢰감 있는 UI/UX 스타일 비주얼
- USP: 모르는 건 모른다고 말하는 AI — 근거 없는 정책은 절대 단정하지 않는 안전장치 내장
- 핵심 메시지(한 문장): "확실하지 않으면, 답하지 않습니다." / "If we're not sure, we don't answer."

---

## 2. 캠페인 목표 / 핵심 메시지

- 캠페인 목표: 인지 — CS 담당자들에게 "리아"라는 솔루션의 존재와 핵심 차별점(근거 없는 확답을 하지 않는다)을 각인
- 핵심 메시지: 확실하지 않으면, 답하지 않습니다. (영상 내 표기는 영어로 통일: "If we're not sure, we don't answer.")

---

## 3. 제작 사양 / 공통 설정

- 화면 비율: 9:16 세로형 (숏폼/릴스 표준)
- 인물 처리: 인물 없이 모니터 화면 중심 (씬 간 캐릭터 일관성 변수 제거)
- 화면 내 텍스트 언어: 영어로 통일 (이미지 생성 시 한글 깨짐 방지 + 톤 일관성 + 글로벌 SaaS 제품 포지셔닝)
- 일관성 유지 전략: 씬 1 확정본 이미지를 레퍼런스로 올려둔 채 후속 씬을 생성 (동일 스타일 고정 → 재생성 횟수 절감)

### 최종 영상 파일 정보

- 파일명: ria_short.mp4 (리아 브랜드 광고)
- 길이: 약 15초
- 해상도: 1080 x 1920 (9:16 세로)
- 프레임레이트: 30fps

```
A vertical 9:16 video frame. Clean, minimal UI/UX aesthetic. Close-up of a computer
monitor screen (no human faces, only screen content). Soft studio lighting, subtle
```

## 4. 씬 구성 (총 약 15초 / 기승전결 구조)

### 씬 1 (Intro) — 문제 제시

- 씬 길이(초): 약 4
- 목표 메시지: 근거 없는 확답이 컴플레인을 키운다
- 화면 구성: "CustHub Support Desk"(가상 CS 툴) 받은편지함 클로즈업 — 환불/로그인 문의가 폭주(542 Unread, 큰 빨간 알림 강조)하고, 성의 없는 미확인 답변("hi im sorry your having trouble... -Support Team")이 작성된 상태에서 커서가 Send 버튼 바로 위에 놓임 / 텍스트 있음
- 내레이션(영문 카피): "One careless reply can cause bigger problems."
- 사용 도구 및 목적:
  - 이미지 생성: 나노바나나(Gemini) — 받은편지함 폭주 장면 키비주얼
  - 비디오 변환: Google Flow (Veo) — 받은편지함 스크롤 + 커서가 Send로 향하는 미세 모션
  - 오디오(내레이션): CapCut TTS — 영어 음성
- 입력 프롬프트(원문): `[공통 스타일 앵커] + The monitor shows a customer service inbox overflowing with unread inquiry messages (Refund request, Help, Waiting for response) piling up, high unread count. A rushed, careless reply is half-typed with the mouse cursor hovering right over the "Send" button. Tense, chaotic mood.`
- 출력 결과 요약: 다급하게 미확인 답변을 보내려는 순간의 긴장감 있는 비주얼 확보
- 결과 파일명: scene01_inbox.mp4

### 씬 2 — 전환

- 씬 길이(초): 약 3
- 목표 메시지: 리아가 보내기 직전 상황을 멈춰 세운다
- 화면 구성: 답변을 보내기 직전, 화면 전체가 차분한 톤으로 정리되며 중앙에 "Policy check required" 안내 카드가 떠오름 / 텍스트 있음
- 내레이션(또는 카피): (없음 — 화면 전환과 BGM으로 전달)
- 사용 도구 및 목적:
  - 이미지 생성: 나노바나나(Gemini) — 씬 1 화면을 레퍼런스로 톤 전환 + 중앙 안내 카드
  - 비디오 변환: Google Flow (Veo) — 톤이 가라앉고 카드가 페이드인되는 모션
- 입력 프롬프트(원문): `[씬 1 이미지를 레퍼런스로 업로드] Keep this exact same monitor screen and layout, but show the moment right before clicking send: the entire screen calmly transitions into a clean cool tone, and a centered notification card gently appears in the middle with the English text "Policy check required". The chaotic inbox fades into the background. Calm, reassuring mood. Vertical 9:16.`
- 출력 결과 요약: 전송 직전 멈춰 세우는 차분한 전환 비주얼 확보
- 결과 파일명: scene02_pause.mp4

- 씬 길이(초): 약 4

- 내레이션(또는 카피): (없음 — 화면과 BGM으로 전달)

  - 이미지 생성: 나노바나나(Gemini) — 완성된 메일 UI + 체크마크 키비주얼
  - 비디오 변환: Google Flow (Veo) — 체크마크가 나타나며 빛나는 모션

## 6. 프롬프트 수정 전/후 기록 (씬 1)

### 기록 ① 구도/연출 (인물 중심 → 모니터 중심)

- 수정 전 의도: "다급한 CS 담당자가 답변을 보내는 장면"을 단순하게 표현
  ```
  rushed customer service agent typing reply, office desk, computer screen
  ```
- 문제: "그냥 사무실에서 일하는 사람"으로만 보이고, 다급함·미확인 답변의 긴장감이 드러나지 않음. 모니터 내용도 거의 안 보임.
- 수정 후 변경:
  ```
  close-up computer monitor screen, customer service inbox overflowing with messages,
  rushed unverified reply being typed, cursor about to click send, tense atmosphere, cinematic 8k
  ```
- 수정 이유(의도): 모니터 화면 클로즈업으로 시점을 좁혀 "쌓인 문의"와 "막 보내려는 답변"을 명확히 노출. "about to click send"로 "보내기 직전" 타이밍 강조.
- 결과 변화: 화면 구성이 인물 중심 → 모니터/UI 중심으로 전환.

### 기록 ② 소스 적합성·비율·인물·가독성 보정

- 1차 결과 문제: ①실제 Gmail 스크린샷에 가까워 "AI 생성 소스" 제약에 부적합 ②인물(어깨/얼굴)이 크게 포함 ③가로 16:9로 생성 ④일부 메뉴 텍스트 깨짐.
- 수정 방향: 실제 Gmail UI를 "참고 레퍼런스"로만 활용하고, AI가 새로운 가상 CS 툴("CustHub Support Desk")을 생성하도록 변경. 동시에 9:16 세로 + 인물 제거 + 텍스트 가독성 확보 요청.
- 결과 변화: 저작권/제약 문제 해소(AI 생성 가상 UI), 세로 비율 전환, 인물 제거, 텍스트 가독성 확보 → 씬 1 확정본 채택. (이후 1080p로 재생성하여 4개 씬 해상도 통일)

## 7. 제작 회고 (요약)

- 영상 AI는 "화면 속 글자가 다른 글자로 바뀌는" 변화를 안정적으로 만들지 못함 → 각 씬을 "정지 이미지 + 미세 모션"으로 생성하고, 내용 전환은 편집(CapCut)에서 컷/트랜지션으로 처리하는 전략을 채택.
- 크레딧 절약을 위해 이미지 단계에서 4개 씬을 모두 확정한 뒤 영상 변환을 진행.
- 화면 텍스트를 영어로 통일하여 AI 이미지의 한글 렌더링 실패 위험을 회피하고 톤 일관성을 확보.
