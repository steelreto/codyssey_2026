# LLM 모델 비교·선정 보고서
## CS 환불/교환 컴플레인 응대 메일 작성 과업

---

## 1. 비교 목적

동일한 시스템 프롬프트(페르소나 "리아" + 출력 형식/안전장치/사실 처리 규칙)와 동일한 입력을 3종 이상의 LLM에 입력하여, **프롬프트 설계가 동일할 때 모델 간 결과 품질이 어떻게 달라지는지**를 정확성(환각 방지)·형식 준수·자연스러움·안전장치 작동·비용 측면에서 비교하고, 본 업무에 가장 적합한 모델을 선정한다.

---

## 2. 비교 대상 모델 및 환경 기록

| 모델 | 요금제 | 채널 | 사용 날짜 | 비고 |
|---|---|---|---|---|
| Claude | Sonnet 4.6 | Pro | 웹 | 2026-06-27 |
| Gemini | Flash 3.5 | Google AI Pro | 웹 | 2026-06-27 | 
| ChatGPT | GPT-5.5 Instant | Free | 웹 | 2026-06-27 |

## 3. 비교에 사용한 동일 입력

세 모델 모두 아래 ①~⑤를 **같은 대화창에서 순서대로** 입력하여 단기(①②)와 장기(10턴, ①~⑤) 결과를 모두 확인했다.

1 시스템 프롬프트 + 입력 A (정상 케이스)
```
(시스템 설계 문서 4번의 시스템 프롬프트 전문)

이제 아래 입력을 바탕으로 메일을 작성해주세요.

[고객 문의] 배송이 너무 늦어져서 환불 요청
[주문 정보] 주문번호 #C11111, 가전제품, 구매일 2026-06-10
[처리 결과] 승인
[적용 정책] 배송지연 환불은 영업일 기준 5일 내 처리, 배송비 회사 부담
[톤] 정중
```

2 입력 B (환각 트랩 — 정책 미제공)
```
이번엔 다른 고객 문의입니다. 같은 규칙으로 작성해주세요.

[고객 문의] 환불 시 배송비도 같이 환불해주나요?
[주문 정보] 주문번호 #C11111
[처리 결과] (없음)
[적용 정책] (제공되지 않음)
[톤] 정중
```

3 늦은 정보 제공 + 재작성 요청 (5턴)
```
참고로 이 고객이 환불 대신 교환을 원할 수도 있어서 알려줄게.
교환도 환불과 동일하게 영업일 기준 5일 내 처리 가능해.
이 내용까지 반영해서 첫 번째 케이스(주문번호 #C11111) 메일을 다시 작성해줘.
```

4 조건 변경: 톤 + 호칭 (7턴)
```
좋아, 그런데 지금부터는 톤을 좀 더 단호하게 바꿔주고,
고객 호칭은 '고객님' 대신 '신청인'으로 통일해줘.
```

5 문맥 유지 검증: 최종 통합 (9턴)
```
지금까지 반영된 내용(환불 승인, 5일 처리, 배송비 회사부담,
교환 가능, 단호한 톤, '신청인' 호칭) 전부 포함해서
최종본으로 다시 한번 정리해서 보내줘.
```

1~5 까지가 모델당 정확히 10턴(사용자 5턴 + AI 5턴)에 해당한다.

## 4. 평가 축 및 점수표 (1~5점)

| 평가 축 | Gemini (Flash · Pro 요금제) | Claude (Sonnet 4.6 · Pro 요금제) | ChatGPT (Free 요금제) |
|---|---|---|---|
| 1 정확성(환각 방지)   | 3 | 3 | **5** |
| 2 형식 준수          | 5 | 3 | 4 |
| 3 한국어 자연스러움   | 5 | 4 | 5 |
| 4 안전장치 작동       | 5 | 5 | 5 |
| 5 문맥 유지(10턴, Pass/Fail) | Pass | Pass | Pass |
| 6 비용/요금제 | 유료 (Pro) | 유료 (Pro) | **무료** |
| **합계 (1~4, 20점 만점)** | 18 | 15 | **19** |

> 참고: ①②③④ 점수는 10턴 전체(① ~ ⑤ 입력)를 반영한 최종 점수이며, 입력 A·B(①②)만 봤을 때의 1차 인상과는 차이가 있다. 예를 들어 정확성은 1차 인상에서 Gemini 4점·Claude 5점이었으나, 5~9턴에서 반복되는 환각 패턴이 발견되어 둘 다 3점으로 하향 조정되었다. 이는 **짧은 테스트보다 10턴 이상의 장기 대화 테스트가 모델의 실제 신뢰도를 더 정확히 드러낸다**는 것을 보여준다.

## 5. 모델별 결과 요약

### Gemini
- 입력 A·B 단계에서는 형식·안전장치·한국어 표현 모두 우수했음.
- 다만 입력 A에서 "결제사 사정에 따라 실제 환불 반영일까지는 시일이 조금 더 소요될 수 있다"는, 적용 정책에 없는 문구를 임의로 추가함.
- 이 문구가 6턴·8턴·10턴에서도 거의 동일하게 **반복 등장** — 단발성 실수가 아니라 고정된 환각 패턴으로 판단됨.
- 호칭("신청인") 통일은 누락 없이 완벽하게 처리했으나, "단호한 톤" 요청에 대한 반영은 약한 편(격식은 높아졌지만 어조 자체는 크게 단호해지지 않음).

### Claude
- 입력 A·B 단계에서는 근거 없는 내용 추가가 없어 정확성이 가장 높게 평가됐었음.
- 그러나 `[상황 요약]` `[처리 결과 및 조치]` `[근거(정책)]` 같은 **내부 구조 라벨이 고객 발송용 메일 본문에 그대로 노출**되는 형식 문제가 반복됨(8턴에서는 일부 라벨이 누락되어 구조가 불안정해지기도 함).
- 8턴(톤·호칭 변경)에서 "회신이 없을 경우 환불로 진행됨을 안내드립니다"라는, **입력에 전혀 없던 새로운 정책을 임의로 창조**하여 10턴까지 그대로 재사용함 — 이는 단순 안내 문구 차원을 넘어 실제 회사 정책과 다른 약속을 고객에게 전달할 수 있는 더 심각한 환각으로 판단됨.
- 호칭 통일은 두 번째 테스트(10턴 버전)에서는 누락 없이 처리됨. 톤 변화는 세 모델 중 가장 뚜렷하게(사무적·건조하게) 반영됨.

### ChatGPT (Free)
- 입력 A·B부터 10턴 종료까지 **근거 없는 내용 추가가 단 한 건도 발견되지 않음** — 3개 모델 중 유일하게 환각이 0건.
- 형식은 라벨 노출 없이 자연스러운 문장형으로 구성되어 가독성이 좋음.
- 다만 7턴(톤·호칭 변경 지시)에 대해, 즉시 수정된 메일을 작성하지 않고 "적용할 규칙을 다음 메일부터 반영하겠다"는 확인성 답변으로 한 턴을 더 소요함 — 시스템 프롬프트의 "최종 출력은 완성된 메일" 원칙에서 다소 벗어난 형식적 결함.
- "단호한 톤" 반영은 세 모델 중 가장 약한 편이며, 무료 요금제로 진행되었음에도 정확성·안전장치 측면에서는 가장 우수했음.

## 6. 최종 선정 결론

본 업무(CS 환불/교환 메일 작성)에서 가장 중요한 평가 축은 **정확성(환각 방지)**인데, ChatGPT만이 10턴 전체에서 근거 없는 내용을 단 한 차례도 추가하지 않아 가장 안정적이었다. Gemini와 Claude는 각각 "결제사 사정으로 지연 가능", "무응답 시 자동 환불 진행"과 같이 한 번 만들어낸 근거 없는 문구를 이후 턴에서도 반복적으로 재사용하는 패턴을 보였는데, 특히 Claude가 만들어낸 가상의 정책은 실제 발송 시 회사 정책과 다른 약속을 고객에게 전달할 위험이 있어 더 심각한 사례로 판단된다. 형식과 자연스러움 면에서도 ChatGPT는 Claude처럼 내부 라벨이 본문에 노출되는 문제 없이 매끄러운 메일을 작성했다. 무엇보다 ChatGPT는 무료 요금제임에도 이 결과를 냈다는 점에서, "모델 등급/비용보다 프롬프트 설계가 결과 품질을 더 크게 좌우한다"는 이번 미션의 핵심 메시지를 가장 잘 보여주는 근거가 된다. 따라서 본 과업에는 **ChatGPT**를 최종 선정한다.

다만 ChatGPT 역시 톤 변경 시 즉시 결과물을 출력하지 않고 확인성 응답을 거치는 작은 형식 결함이 있었으므로, 실제 도입 시에는 "조건 변경 지시를 받으면 별도 확인 없이 즉시 수정된 결과물을 출력하라"는 규칙을 시스템 프롬프트에 한 줄 추가하는 보완이 필요하다.
